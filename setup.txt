==============================================================================
 RepoRadar — Local Setup
==============================================================================

RepoRadar is a Next.js full-stack app. Embeddings run locally (free, no key).
The only paid service is the LLM via OpenRouter — and you can skip even that by
leaving NO_LLM_MODE=true.

------------------------------------------------------------------------------
0) PREREQUISITES
------------------------------------------------------------------------------
- Node.js 20+            (you have v24)
- pnpm                   (install once: `npm install -g pnpm`)
- Docker Desktop         (for the local Postgres + pgvector database)
                         Get it from https://www.docker.com/products/docker-desktop
                         NOTE: Docker is NOT currently installed on this machine.
                         Install it (and start it) before step 2.

------------------------------------------------------------------------------
1) CONFIGURE ENVIRONMENT
------------------------------------------------------------------------------
Copy the example env file and edit it:

    copy .env.example .env          (Windows)
    # or:  cp .env.example .env     (macOS/Linux)

Then open `.env` and set these. The most important one is the LLM API key:

    # ---- THE LLM API KEY (OpenRouter) ----
    OPENROUTER_API_KEY=sk-or-...    <-- get it at https://openrouter.ai  ->  Keys
    OPENROUTER_MODEL=deepseek/deepseek-chat
    NO_LLM_MODE=false               <-- set false ONLY after the key is set;
                                        leave true to run completely free.

    # ---- GitHub (strongly recommended) ----
    GITHUB_TOKEN=ghp_...            <-- a classic Personal Access Token with
                                        read-only public access. Without it the
                                        GitHub Search API throttles to ~10/min.
                                        Create at: github.com -> Settings ->
                                        Developer settings -> Personal access tokens.

    # ---- Database (already matches docker-compose.yml; usually no change) ----
    DATABASE_URL=postgresql://reporadar:reporadar@localhost:5432/reporadar?schema=public

    # ---- Tuning ----
    FUNNEL_TOP_N=15                 <-- how many repos the AI scores per search (<=20)

    # ---- Filesystem compatibility ----
    REPORADAR_EXFAT_MODE=false      <-- leave false on NTFS/Linux/macOS. Set true only
                                        when the repo is on an exFAT drive.

Notes on the API key / payment:
  * Embeddings are LOCAL (Transformers.js, all-MiniLM-L6-v2). They need NO key and
    cost nothing. The ~25-90 MB model downloads once on first use, then is cached.
  * OpenRouter credits are topped up by card/crypto — NOT WeChat/Alipay.
  * If you don't want to pay anything yet: keep NO_LLM_MODE=true. The app runs the
    full pipeline and ranks repos by the deterministic funnel (no AI scores).

------------------------------------------------------------------------------
2) START THE DATABASE
------------------------------------------------------------------------------
    docker compose up -d

This starts Postgres 16 with the pgvector extension on localhost:5432.

------------------------------------------------------------------------------
3) INSTALL DEPS + APPLY THE SCHEMA
------------------------------------------------------------------------------
    pnpm install
    pnpm db:deploy        (applies prisma/migrations: tables + vector ext + index)

    # If you prefer to sync the schema without migration history:
    #   pnpm db:push

------------------------------------------------------------------------------
4) RUN
------------------------------------------------------------------------------
    pnpm dev

Open http://localhost:2000

Health check (DB + pgvector):   http://localhost:2000/api/health
Expected when DB is up:         {"status":"ok","db":true,"pgvector":true}

------------------------------------------------------------------------------
5) USE IT
------------------------------------------------------------------------------
- Type what you want, e.g. "maintained TypeScript markdown editor library".
- Optionally open Filters (language, license, project type, recency, small repos).
- Watch the progress bar; results stream in as Fit / Future / Underrated scores
  with a score breakdown, "Best of" rails, an Underrated section, and compare.
- Click a repo to see its detail page with radar charts and README evidence.

------------------------------------------------------------------------------
TROUBLESHOOTING
------------------------------------------------------------------------------
- /api/health says db:false  -> Docker isn't running, or `docker compose up -d`
  hasn't been run, or DATABASE_URL is wrong.
- Few/no results              -> add a GITHUB_TOKEN (rate limits), broaden the
  prompt, or relax filters.
- No AI explanations          -> NO_LLM_MODE is true, or OPENROUTER_API_KEY is unset.
- Search stays loading        -> check http://localhost:2000/api/health first,
  then inspect logs/pipeline.log and the terminal running `pnpm dev`. The home
  page pre-warms /api/search; each POST /api/search logs a requestId, queued
  searchQueryId, browser user-agent/referrer, background pipeline launch, and
  pipeline stage timings. The browser console also logs route warm-up, POST
  completion, navigation, and result polling state. Fresh searches usually
  take about 40 seconds.
- You see `ChunkLoadError: Loading chunk app/layout failed` -> the browser is on
  a stale dev bundle. Hard reload the page or restart `pnpm dev` after clearing
  `.next`; the root layout is server-only now, and React Query lives only on the
  results route.
- Search button says "Searching..." but never opens results -> stop `pnpm dev`,
  remove `.next`, start `pnpm dev` again, then reload http://localhost:2000.
  This clears stale Next dev manifests after a bad hot reload.
- pnpm install symlink errors -> if the repo is on exFAT, keep `nodeLinker:
  hoisted` in pnpm-workspace.yaml and set REPORADAR_EXFAT_MODE=true. On NTFS,
  Linux, macOS, or WSL, leave REPORADAR_EXFAT_MODE=false.
- `pnpm build` fails with EISDIR/readlink/junction errors -> that is usually an
  exFAT filesystem limitation, not a code bug. Set REPORADAR_EXFAT_MODE=true
  for exFAT development, or run production builds on Linux/Railway, NTFS, or WSL.

------------------------------------------------------------------------------
COMMANDS REFERENCE
------------------------------------------------------------------------------
    pnpm dev           start dev server (http://localhost:2000)
    pnpm build         production build (run on Linux/NTFS, not exFAT)
    pnpm start         run the production build
    pnpm typecheck     TypeScript check
    pnpm db:deploy     apply migrations
    pnpm db:push       sync schema without migrations
    pnpm db:studio     open Prisma Studio (browse the DB)
    docker compose up -d / down     start / stop Postgres
